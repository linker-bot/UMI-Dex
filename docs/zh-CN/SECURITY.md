# 安全政策

**UMI-Dex** 是由 **Linkerbot** 维护的灵巧手遥操作数据采集栈，当前采用 ROS1 录制与 Python 分析工具并行架构。本项目采用 [Apache 2.0 许可证](../../LICENSE) 授权。

**其它语言版本：**见 [文档索引](../README.md)（含英文 `en` 等）。

---

## 受支持版本

| 版本 | 支持状态 | 说明 |
|------|----------|------|
| **v1.0.0** | 是 | 已在 **ROS1 Noetic** + Python 工具（`uv`）上测试 |

旧版或未发布版本可能不再获得安全更新，建议使用最新受支持版本。

---

## 如何报告漏洞

**请勿**通过公开的 GitHub Issue、Discussion 或 Pull Request 报告安全问题，以免在修复发布前暴露风险。

**请通过邮件私下报告安全问题：**

**[helloworld@linkerbot.cn](mailto:helloworld@linkerbot.cn)**

请在邮件主题中包含 `[SECURITY]` 或 `[安全]`，以便我们优先处理。

---

## 报告中应包含的内容

为便于我们评估与修复，请尽量包含：

1. **描述** — 问题是什么、出现在何处（如已知请写明组件、文件或节点）。
2. **影响** — 可能影响（如数据泄露、设备控制、拒绝服务等）。
3. **复现步骤** — 在可安全分享的前提下，提供最小复现步骤或概念验证。
4. **环境** — 操作系统、ROS 发行版（如 Noetic）、Python 版本、UMI-Dex 版本及相关硬件。
5. **联系方式** — 便于我们跟进（留邮箱即可）。
6. **披露时间偏好** — 若对协调披露时间有要求，请一并说明。

---

## 响应时间预期

以下为**目标**，非承诺；严重性与复杂度可能影响实际节奏。

| 阶段 | 目标 |
|------|------|
| **首次确认收到** | **5 个工作日内** |
| **进度更新** | 调查过程中定期同步；若问题未关闭，约 **每 14 天** 至少一次 |
| **修复与公告** | 视严重程度而定；我们力求在就绪时发布修复并协调披露 |

若在上述窗口内未收到确认，可向同一邮箱礼貌地跟进一封邮件。

---

## 披露政策

1. **先私下协调** — 在可行情况下，我们会在修复或缓解措施可用前与报告者非公开协作。
2. **协调披露** — 除非法律要求或用户安全急需，请在我们有合理时间处理问题前避免公开披露。
3. **致谢** — 经您同意后，我们可能在发行说明或公告中致谢。
4. **范围外** — 关于第三方依赖的问题可能转交上游；我们仍会确认收到。

---

## 安全范围

本政策下的安全审查与响应范围包括本仓库内属于 UMI-Dex 栈的组件，例如：

| 领域 | 示例 |
|------|------|
| **ROS1 节点** | `ros/` 下节点逻辑、参数、话题、launch 流程与录包行为。 |
| **串口通信** | 与灵巧手、控制器或相关设备的串口接口与数据处理。 |
| **数据采集脚本** | 录制或导出遥操作/传感器数据的脚本与流水线。 |

**通常不在**本项目安全流程范围内（仍可发邮件以便转交）：与本项目补丁无关的第三方 ROS 包、纯物理篡改假设、以及仅存在于上游 ROS 或操作系统的问题——除非与我们的代码以可明确利用的方式交互。
